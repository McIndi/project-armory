# envoy_gateway

Deploys Envoy Gateway as the Kubernetes Gateway API edge, replacing
ingress-nginx (retired upstream). Owns the `GatewayClass`, the `Gateway`
(listeners + edge TLS), the `EnvoyProxy` data-plane/telemetry configuration,
and the trace-context trust boundary. Per-workload `HTTPRoute`s stay with the
owning roles (keycloak, headlamp, openbao, delve).

## Trace-context trust boundary

The gateway is the trust boundary for trace identity: inbound `traceparent`,
`tracestate`, `baggage`, and `b3`/legacy headers are stripped by a
`ClientTrafficPolicy` with `earlyRequestHeaders` ã before the tracing
decision ã on all external traffic, uniformly. The gateway mints a fresh
random trace-id and its span is the trace root; the sampling decision (100%)
is made at the gateway, never from an inbound `sampled` flag. Internal
propagation (gateway ã Keycloak ã OpenBao ã agents) is untouched.

Forensics: the inbound `traceparent` is copied to `x-external-traceparent`
before the strip and recorded as span attribute `external.traceparent` on the
gateway root span ã correlation without trust. The readiness trace-boundary
stage asserts the strip, the fresh mint, and the forensic capture end to end.

Health probes are kubelet-direct (never traverse the gateway) so they generate
no traces; the gateway's own admin/ready endpoints are untraced.

## Service exposure

k3s runs with both `traefik` and `servicelb` disabled. The Envoy Service is
`ClusterIP` patched with the node address as an `externalIP`, so kube-proxy
binds 443 (and 80 under `ingress_http_policy: redirect-only`) on the node.
In-cluster callers that must reach the public issuer host resolve it to the
Envoy Service clusterIP via
`common/tasks/lookup_gateway_service.yml` (label-based; the Service name is
generated by Envoy Gateway).

## TLS

One consolidated edge certificate (`armory-tls`, OpenBao external
ClusterIssuer) covers the apex domain, the workload subdomains, and the node
IP SAN. It lives in the gateway namespace ã listener `certificateRefs` are
same-namespace. Backend connections re-encrypt: each workload role applies a
`BackendTLSPolicy` validating the internal CA (trust-manager
`openbao-ca-bundle` ConfigMap) against the service FQDN.

## Tracing backend

A minimal OTLP collector (debug exporter) receives gateway spans; replace
`envoy_gateway_otel_collector_*` / the EnvoyProxy `backendRefs` when a real
trace store lands. Disable with `envoy_gateway_tracing_enabled: false`.

## Deferred

Authenticated machine callers (mTLS / client-credential service traffic) may
honor inbound trace context end-to-end via a dedicated mTLS listener gated on
the auth signal ã permitted by policy, not yet implemented.
